---
title: Tables
---

Tables define data storage schemas for your agent. They provide structured storage that's automatically synced with Botpress Cloud and accessible throughout your agent.

## Creating a table

Create a table definition in `src/tables/`:

```typescript
import { Table } from "@botpress/runtime";

export default new Table({
  name: "users",
  description: "User information table",
  schema: {
    id: "string",
    name: "string",
    email: "string",
    createdAt: "timestamp",
  },
});
```

## Table schema

Define your table schema using simple type definitions:

```typescript
export default new Table({
  name: "orders",
  schema: {
    id: "string",
    userId: "string",
    items: "array",
    total: "number",
    status: "string",
    createdAt: "timestamp",
    updatedAt: "timestamp",
  },
});
```

Supported types:

- `"string"` - Text data
- `"number"` - Numeric data
- `"boolean"` - Boolean values
- `"timestamp"` - Date/time values
- `"array"` - Array of values
- `"object"` - Nested object structures

## Using tables

Tables are automatically created and synced when you deploy. Access them through the Botpress client:

```typescript
// In a conversation or workflow
export default new Conversation({
  channel: "*",
  handler: async ({ execute, context }) => {
    // Create a row
    await context.client.tables.upsertRows({
      table: "users",
      rows: [{
        id: "user-123",
        name: "John Doe",
        email: "john@example.com",
        createdAt: new Date(),
      }],
    });

    // Find rows
    const users = await context.client.tables.findRows({
      table: "users",
      filter: {
        email: "john@example.com",
      },
    });

    await execute({
      instructions: "You are a helpful assistant.",
    });
  },
});
```

## Table operations

The Botpress client provides table operations:

- `createRows` - Create new rows
- `updateRows` - Update existing rows
- `upsertRows` - Create or update rows
- `findRows` - Query rows with filters
- `deleteRows` - Delete rows

<Tip>
Tables are automatically synced with Botpress Cloud. When you deploy your agent, table schemas are created or updated to match your definitions.
</Tip>
