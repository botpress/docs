The `Webchat` component is a batteries-included implementation of the Webchat window. It contains most of what you'd expect when [injecting Webchat](/webchat/get-started/quick-start).

    ``` jsx App.jsx
    import { Fab, Webchat } from '@botpress/webchat'
    import { useState } from 'react'

    function App() {
    const [isWebchatOpen, setIsWebchatOpen] = useState(false)
    const toggleWebchat = () => {
        setIsWebchatOpen((prevState) => !prevState)
    }
    return (
        <>
        <Webchat
            clientId="$CLIENT_ID$" // Your client ID here
            style={{
            width: '400px',
            height: '600px',
            display: isWebchatOpen ? 'flex' : 'none',
            position: 'fixed',
            bottom: '90px',
            right: '20px',
            }}
        />
        <Fab
            onClick={() => toggleWebchat()}
            style={{
            position: 'fixed',
            bottom: '20px',
            right: '20px',
            width: '64px',
            height: '64px'
            }}
        />
        </>
    )
    }

    export default App
    ```
    </CodeGroup>
    </Tab>
</Tabs>
</Panel>

<Note>
    The `Webchat` component doesn't use the configuration set in your bot's [Dashboard](/learn/get-started/dashboard/bot/webchat). Instead, configure it using the [configuration](#configuration) prop.
</Note>

<Warning>
  The `<Webchat>` component is incompatible with the [`useWebchat` hook](/webchat/react-library/hooks/use-webchat-client)â€”using them together will cause issues and unexpected behaviour.

  If you need to use the hook, make sure you're [manually building Webchat](/webchat/react-library/get-started#option-2%3A-build-webchat-manually).
</Warning>

## Props

<ResponseField
    name="clientId"
    type="string"
    required
>
    Your bot's [client ID](/webchat/react-library/get-started#obtain-your-client-id).
</ResponseField>

<ResponseField
    name="storageKey"
    type="string"
>
    Custom key for Webchat's local storage ([if your bot's configuration uses local storage](#param-storage-location))
</ResponseField>

<ResponseField
    name="user"
    type="UserCredentials"
>
    User to load Webchat with.
    <Expandable>
        <ResponseField
            name="userId"
            type="string"
        >
            The user's ID.
        </ResponseField>
        <ResponseField
            name="userToken"
            type="string"
        >
            The user's token.
        </ResponseField>
    </Expandable>
</ResponseField>

<ResponseField
    name="conversationId"
    type="string"
>
    Conversation ID to load Webchat with.
</ResponseField>

<ResponseField
    name="configuration"
    type="Configuration"
>
    Configuration for Webchat's appearance and style.

    <Expandable>
        <ResponseField name="botName" type="string">
            Name of the bot displayed in the header.
        </ResponseField>
        <ResponseField name="botAvatar" type="string">
            URL to the bot avatar image.
        </ResponseField>
        <ResponseField name="botDescription" type="string">
            Description text shown in the Webchat header.
        </ResponseField>
        <ResponseField name="fabImage" type="string">
            Image for the [Fab](/webchat/react-library/components/fab).
        </ResponseField>
        <ResponseField name="phone" type="{ title?: string, link?: string }">
            Phone contact information with title and link.
        </ResponseField>
        <ResponseField name="email" type="{ title?: string, link?: string }">
            Email contact information with title and link.
        </ResponseField>
        <ResponseField name="website" type="{ title?: string, link?: string }">
            Website link information with title and URL.
        </ResponseField>
        <ResponseField name="termsOfService" type="{ title?: string, link?: string }">
            Terms of Service link with title and URL.
        </ResponseField>
        <ResponseField name="privacyPolicy" type="{ title?: string, link?: string }">
            Privacy Policy link with title and URL.
        </ResponseField>
        <ResponseField name="composerPlaceholder" type="string">
            Placeholder text for the input composer.
        </ResponseField>
        <ResponseField name="color" type="string">
            Primary theme color in hex format.
        </ResponseField>
        <ResponseField name="radius" type="number">
            Corner radius for UI elements. Can be `0.5` - `4`
        </ResponseField>
        <ResponseField name="themeMode" type="'light' | 'dark'">
            Light or dark mode theme.
        </ResponseField>
        <ResponseField name="variant" type="'solid' | 'soft'">
            Visual style of the chat window.
        </ResponseField>
        <ResponseField name="fontFamily" type="'rubik' | 'inter' | 'ibm' | 'fira'">
            Font family used in Webchat.
        </ResponseField>
        <ResponseField name="showPoweredBy" type="boolean">
            Flag to show or hide the "Powered by Botpress" footer.
        </ResponseField>
        <ResponseField name="additionalStylesheet" type="string">
            Custom inline CSS to apply to the Webchat.
        </ResponseField>
        <ResponseField name="additionalStylesheetUrl" type="string">
            URL pointing to an external stylesheet.
        </ResponseField>
        <ResponseField name="allowFileUpload" type="boolean">
            Flag to enable file uploads in the chat.
        </ResponseField>
        <ResponseField name="storageLocation" type="'localStorage' | 'sessionStorage'">
            Location to store Webchat session data.
        </ResponseField>
        <ResponseField name="footer" type="string">
            Footer content in Markdown.
        </ResponseField>
        <ResponseField name="headerVariant" type="'solid' | 'glass'">
            Style variant of the header.
        </ResponseField>
        <ResponseField name="feedbackEnabled" type="boolean">
            Toggle for user feedback feature.
        </ResponseField>
    </Expandable>
</ResponseField>
