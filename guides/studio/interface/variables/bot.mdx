---
title: Bot variables
---

Bot variables let you store and reuse information **throughout your entire bot**.

Any information you assign to a bot variable will be accessible across all Workflows and conversations. This makes bot variables useful for storing global information about your bot, like:

- Your bot's name
- Your bot's version number
- Endpoints for your bot's API calls

<Warning>
    Bot variables are not encrypted or stored securely. This means **an Autonomous Node can read and display any information you store in a bot variable**.
    
    Don't save sensitive information like API keys or passwords in bot variables — use [**configuration variables**](/guides/studio/interface/variables/configuration) instead.
</Warning>

<Tip>
    Need help understanding variables? Check out our [introduction to variables](/guides/studio/interface/variables/variables).
</Tip>

## Create a bot variable
​
To create a bot variable:

1. Open any Workflow.
2. Select **+** in the **Variables** section at the bottom of the left navigation bar.
3. Choose a **Type** and **Name** for your variable:

![Bot variable](/guides/studio/interface/variables/bot-variable.png)

## Read your variable

Once you've created your variable, you can use `{{bot.variablename}}` or `@bot.variablename` to read it. For example, in a **Text** Card:

![Bot variable in a text card](/guides/studio/interface/variables/bot-variable-example.png)
